# Repository Instructions

## Notebook-first implementation

- Implement project backlog tasks as Jupyter notebooks (`.ipynb`) under `notebooks/`.
- Do not create new Python source or test files (`.py`) for backlog-task implementation.
- Keep each notebook self-contained, with Markdown explaining its purpose and code cells containing both the implementation and its validation checks.
- Execute and verify the notebook before marking its task complete.
- Existing Python files are legacy project infrastructure; do not use them as the pattern for new task work.
